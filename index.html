結合方法の設定

※ このHTML内の外部CSS、外部JS、画像などをすべてインライン展開します。

※ 元の <script type="module"> タグは、インライン化後もモジュールスコープ（遅延実行）を維持するため `type="module"` 属性を残したままインライン展開されます。

インライン化の対象設定

※ 指定されたJSからimport/require文をたどり、自動で解析します。

※ AST（抽象構文木）解析エンジン「Babel Parser」を搭載！複合インポート、aliasインポート、名前付き再エクスポートをサポート！

※ Three.jsなどの外部CDNライブラリや importmap との共存（ネイティブ import フォールバック）に対応！

※ Web Worker環境（globalThis）での動作に完全対応！

※ 制限事項: 1行に複数の変数宣言を並べる `export const a = 1, b = 2;` には対応していません。個別に行を分けてエクスポートしてください。

結合順序の調整 ▲▼ボタンで優先順を変更
各ファイルを変数衝突が起きないように個別の関数に隔離して結合します

右側のテキストエリアに直接コードを貼り付けてください。

ファイルアップロード不要で、単一のコードを極限までゴルフ圧縮します。余計なモジュールラッパーも付与されません。

コードゴルフ化オプション
スコープ内の変数を安全に圧縮。既存の1文字変数は保護され、2フェーズコンパイルでBabelの衝突を防ぎます
デバッグ用のログ出力文を安全に消去（式内の場合はundefinedに置換）
属性クォーテーションの最適除去や小数の頭の0、冗長なマージン指定などをクレンジング
true➔!0、10000➔1e4、単一if文➔&&短絡評価、定数畳み込み、while(true)➔for(;;)、無名関数のアロー化など
文字列内のGLSLシェーダーの空白・コメント除去、Web Worker JSの再帰圧縮
